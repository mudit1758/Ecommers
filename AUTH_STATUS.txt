🔐 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

User Signs Up:
  1. User → /signup
  2. Fills form (name, email, password)
  3. Validates password (min 6 chars)
  4. Confirms password match
  5. Stores in Zustand + localStorage
  6. Redirects to home
  7. Shows profile icon in navbar

User Logs In:
  1. User → /login
  2. Enters email & password
  3. Validates credentials
  4. Loads user from Zustand
  5. Saves to localStorage
  6. Redirects to home
  7. Shows profile icon

User Accesses Pages:
  1. User clicks profile icon
  2. Dropdown shows options
  3. Clicks "My Profile" / "Settings" / etc
  4. Page checks if logged in
  5. If not logged in → Redirects to /login
  6. If logged in → Shows page

User Logs Out:
  1. Clicks profile icon
  2. Clicks "Logout"
  3. Clears user data
  4. Navbar shows "Sign In" / "Sign Up"
  5. Accessing /profile redirects to /login

Issue: "Can't login after signup"
→ Use exact same email and password you signed up with

Issue: "User data not persisting"
→ Check if localStorage is enabled in browser
→ Open DevTools → Application → Look for auth-storage

Issue: "Profile page shows error"
→ Make sure you're logged in first
→ Try going to /login page first

Issue: "Mobile menu doesn't show login options"
→ Click the hamburger menu icon (top right)
→ Auth options should appear

Issue: "Password strength not showing"
→ You need to type in the password field
→ It shows as you type
